# 工数管理システム 設計書

## 1. システム概要

作業者ごとの予定工数・実績工数を管理し、月次・年次での比較分析、メンバーコスト把握を行うWebシステム。

| 項目 | 内容 |
|------|------|
| システム名 | 工数管理システム（kousuuKanri） |
| ライセンス | Apache 2.0 |
| 実行環境 | Docker (docker-compose) |

## 4. データベース設計

### 4.1 テーブル一覧

| テーブル名 | 用途 |
|-----------|------|
| members | メンバーマスタ |
| work_hours | 工数データ（予定・実績） |
| config | システム設定 |

### 4.2 members テーブル

| カラム | 型 | NOT NULL | デフォルト | 説明 |
|-------|-----|---------|-----------|------|
| id | INT UNSIGNED AUTO_INCREMENT | ✓ | — | 主キー |
| code | VARCHAR(50) | ✓ | — | メンバーコード（UNIQUE） |
| name | VARCHAR(100) | ✓ | — | 氏名 |
| unit_cost | DECIMAL(10,2) | ✓ | 0 | 単価（円/時間） |
| active | TINYINT(1) | ✓ | 1 | 在籍フラグ（0=退職） |
| created_at | DATETIME | ✓ | CURRENT_TIMESTAMP | — |
| updated_at | DATETIME | ✓ | CURRENT_TIMESTAMP ON UPDATE | — |

### 4.3 work_hours テーブル

| カラム | 型 | NOT NULL | デフォルト | 説明 |
|-------|-----|---------|-----------|------|
| id | INT UNSIGNED AUTO_INCREMENT | ✓ | — | 主キー |
| member_id | INT UNSIGNED | ✓ | — | members.id 外部キー |
| year | SMALLINT UNSIGNED | ✓ | — | 年（西暦） |
| month | TINYINT UNSIGNED | ✓ | — | 月（1〜12） |
| planned_hours | DECIMAL(6,2) | — | NULL | 予定工数（NULL=未登録） |
| actual_hours | DECIMAL(6,2) | — | NULL | 実績工数 |
| note | VARCHAR(255) | — | NULL | 備考 |
| created_at | DATETIME | ✓ | CURRENT_TIMESTAMP | — |
| updated_at | DATETIME | ✓ | CURRENT_TIMESTAMP ON UPDATE | — |

**ユニーク制約:** `(member_id, year, month)`

> `planned_hours IS NULL` が「予定工数未登録」を表す。締日判定に使用。

### 4.4 config テーブル

| config_key | デフォルト値 | 説明 |
|-----------|------------|------|
| planned_hours_deadline_day | 10 | 予定工数締日（日）|
| fiscal_year_start_month | 4 | 年度開始月（4=4月） |
| currency | JPY | 通貨 |
| company_name | 工数管理システム | システム名称 |

## 5. API設計

ベースパス: `/api/v1`

### 5.1 Config

| メソッド | パス | 説明 |
|---------|------|------|
| GET | /config | 全設定取得 |
| PUT | /config/:key | 単一設定更新（body: `{ value: string }`） |

### 5.2 Members

| メソッド | パス | 説明 |
|---------|------|------|
| GET | /members | メンバー一覧（`?active=1` で在籍のみ） |
| POST | /members | メンバー作成 |
| PUT | /members/:id | メンバー更新 |
| DELETE | /members/:id | メンバー退職（active=0 ソフトデリート） |

### 5.3 Work Hours

| メソッド | パス | 説明 |
|---------|------|------|
| GET | /work-hours | 工数一覧（`?year=&month=`） |
| PUT | /work-hours/:memberId/:year/:month | 工数UPSERT |

### 5.4 Reports

| メソッド | パス | 説明 |
|---------|------|------|
| GET | /reports/monthly | 月次レポート（`?year=&month=`） |
| GET | /reports/annual | 年次レポート（`?fiscal_year=`） |

**月次レポートレスポンス例:**

```json
      "planned_cost": null, "actual_cost": null

```

### 5.5 Import

| メソッド | パス | 説明 |

## 6. CSVフォーマット仕様

### 6.1 ファイル仕様

| 項目 | 仕様 |
|------|------|
| 文字コード | UTF-8（BOM付き可） |
| 区切り文字 | カンマ（,） |
| ヘッダー | 1行目必須 |
| 最大ファイルサイズ | 5MB |

### 6.2 カラム定義

| カラム名 | 必須 | 説明 |
|---------|------|------|
| member_code / メンバーコード | ※ | members.code と一致 |
| member_name / メンバー名 | ※ | members.name と一致（codeがない場合） |
| year / 年 / 年度 | ✓ | 西暦（例: 2026） |
| month / 月 | ✓ | 月（1〜12） |
| type / 種別 | ✓ | `planned`/`予定` または `actual`/`実績` |
| hours / 工数 | ✓ | 工数（小数可、0以上） |
| note / 備考 | — | 任意メモ |

※ member_code または member_name のどちらか一方が必須

### 6.3 サンプル

```csv
member_code,year,month,type,hours,note
M001,2026,4,planned,160,
M001,2026,4,actual,152.5,残業あり
M002,2026,4,planned,168,
M003,2026,4,actual,170,
```

### 6.4 インポート動作

- 同一`(member_id, year, month)`の既存レコードは UPSERT（後勝ち）
- メンバーが見つからない行はスキップ（エラーログに記録、処理は継続）
- レスポンス: `{ total, imported, skipped, errors: [{row, reason}] }`

## 8. 締日アラート仕様

| 条件 | 表示 |
|------|------|
| `today > deadline_day` かつ `planned_hours IS NULL` | メンバー行・テキストを赤色表示 |
| 対象月が過去月（今月より前） | 同上（締日に関わらず赤色） |

**判定ロジック（サーバーサイド）:**
```
isPastDeadline =
  year < today.year
  OR (year == today.year AND month < today.month)
  OR (year == today.year AND month == today.month AND today.day >= deadlineDay)
```

## 9. 起動方法

### 初回起動

```bash
cp .env.example .env
# 必要に応じて .env のパスワードを変更
docker-compose up --build
```

ブラウザで `http://localhost` を開く。

### 通常起動

```bash
docker-compose up
```

### 停止・データ削除

```bash
docker-compose down        # コンテナ停止（データ保持）
docker-compose down -v     # コンテナ停止 + DBデータ削除
```

### ローカル開発（Docker不使用）

```bash
# MySQL起動後
# backend
cd backend && npm install && npm run dev

# frontend（別ターミナル）
cd frontend && npm install && npm run dev
# → http://localhost:5173（/api は localhost:3000 にプロキシ）
```
